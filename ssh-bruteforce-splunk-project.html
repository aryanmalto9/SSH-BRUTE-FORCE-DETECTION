🔍 Basic SSH failure detection — Searches all indexes for failed password events. The simplest starting point for SSH brute-force detection.
index=* "Failed password" -- Returns all log events containing "Failed password" -- Each result = one failed SSH login attempt -- Check sourcetype=syslog for Linux auth logs
📊 Count failed attempts per IP — Groups results by source IP and counts attempts. Sorted descending to show highest-volume attackers first.
index=* "Failed password" | stats count by src_ip | sort -count -- stats count by src_ip: groups events by source IP -- sort -count: descending order (most attempts first)
🔎 Extract IP with rex — Uses regex to parse the IP address from raw log text when field extraction isn't automatic. Essential for unstructured logs.
index=* "Failed password" | rex "from (?<src_ip>\d+\.\d+\.\d+\.\d+)" | stats count by src_ip | sort -count -- rex: named capture group extracts IP from log string -- (?<src_ip>...) creates field 'src_ip' dynamically
🚨 Detect brute-force pattern — Core detection logic. Buckets time into 5-minute windows and flags any IP with more than 10 failures — classic brute-force signature.
index=* "Failed password" | rex "from (?<src_ip>\d+\.\d+\.\d+\.\d+)" | bucket _time span=5m | stats count by _time src_ip | where count > 10 -- bucket _time span=5m: groups into 5-min windows -- where count > 10: only show IPs with 10+ failures -- Threshold can be tuned: 5 (strict) to 50 (loose)
📈 Time chart of attempts — Visualize attack waves over time. Perfect for Splunk Dashboard Panel 1 — use Line Chart visualization to see spike patterns.
index=* "Failed password" | timechart count span=1m -- timechart: automatically creates time-series chart -- span=1m: 1-minute buckets (adjust as needed) -- Splunk Dashboard: Visualization → Line Chart
👤 Targeted usernames — Extracts and counts which usernames are being targeted most. Helps identify credential stuffing against specific accounts.
index=* "Failed password" | rex "for (?<user>\w+) from" | stats count by user | sort -count | head 10 -- Extracts username field from "for USERNAME from" -- head 10: top 10 most targeted accounts
⚡ BONUS: Successful login after brute force — The most dangerous pattern. Detects attackers who succeeded after repeated failures — indicates a compromised account!
index=* ("Failed password" OR "Accepted password") | rex "from (?<src_ip>\d+\.\d+\.\d+\.\d+)" | stats count(eval(searchmatch("Failed password"))) as failed count(eval(searchmatch("Accepted password"))) as success by src_ip | where failed > 10 AND success > 0 -- 🚨 CRITICAL: This = confirmed account compromise! -- An IP that failed 10+ times but then succeeded -- Immediately escalate as Severity: CRITICAL

LAB SETUP GUIDE — STEP BY STEP
✓
STEP 1 — INSTALL SPLUNK ENTERPRISE
▶

Download and install Splunk Enterprise on your Splunk server (192.168.1.30). Splunk runs on port 8000 by default.

wget -O splunk.tgz https://download.splunk.com/products/splunk/releases/9.0.0/linux/splunk.tgz
tar -xvzf splunk.tgz && cd splunk/bin
./splunk start --accept-license

Access Splunk Web UI at: http://192.168.1.30:8000

✓
STEP 2 — INSTALL SPLUNK UNIVERSAL FORWARDER (VICTIM)
▶

Install the Universal Forwarder on your Ubuntu victim machine (192.168.1.10). This sends logs to Splunk.

wget https://download.splunk.com/products/universalforwarder/releases/9.0.0/linux/splunkforwarder.tgz
tar -xvzf splunkforwarder.tgz && cd splunkforwarder/bin
./splunk start --accept-license
./splunk add forward-server 192.168.1.30:9997

Enable receiving on Splunk server: Settings → Forwarding and Receiving → New Port → 9997

✓
STEP 3 — FORWARD AUTHENTICATION LOGS
▶

Tell the Universal Forwarder to monitor the SSH authentication log file:

./splunk add monitor /var/log/auth.log
./splunk restart

Verify in Splunk: index=* sourcetype=syslog "sshd"

4
STEP 4 — SIMULATE ATTACK FROM KALI LINUX
▶

On your Kali Linux machine (192.168.1.20), first scan the target then launch the brute force:

Logs will start appearing in /var/log/auth.log and forwarded to Splunk.

5
STEP 5 — BUILD SPLUNK DASHBOARD
▶

In Splunk Web UI, go to Dashboards → Create New Dashboard and add these panels:

Run the brute-force detection SPL query, then: Save As → Alert
